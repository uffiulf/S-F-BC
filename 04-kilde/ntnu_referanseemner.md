## IT2901 — Informatikk prosjektarbeid II (Bachelor)

**✅ Bekreftet:**

| Egenskap | Verdi |
|:---------|:------|
| **Studiepoeng** | 15 SP |
| **Nivå** | Tredjeårsemne (bachelor) |
| **Undervisning** | Vår 2027, Trondheim, engelsk |
| **Gruppestørrelse** | 5–7 studenter |
| **Eksterne kunder** | Ja — tett samarbeid påkrevd |
| **Vurdering** | Bokstavkarakter. Gruppebasert oppgave (100 %) |

**Organisering:**
- Prosjekter **velges og tildeles av instituttet**
- Seminarer, jevnlige statusrapporter, møter med veileder og kunde
- Midtveisrapport og sluttrapport
- Hele gruppen får normalt samme karakter; individuell vurdering ved dokumentert manglende innsats

**Forkunnskaper:** IT1901, TDT4100 (OOP), TDT4140 (Programvareutvikling). Kun for BIT eller MLREAL-studenter.

**Emneansvarlig:** Monica Divitini, IDI

**Kilde:** [ntnu.no/studier/emner/IT2901](https://www.ntnu.no/studier/emner/IT2901)

## TDT4290 — Kundestyrt prosjekt / Customer Driven Project (Master)

**✅ Bekreftet:**

| Egenskap | Verdi |
|:---------|:------|
| **Studiepoeng** | 15 SP |
| **Nivå** | Høyere grads nivå (master) |
| **Undervisning** | Høst 2026, Trondheim, engelsk |
| **Gruppestørrelse** | 6–8 studenter |
| **Eksterne kunder** | Ja — hver gruppe får tildelt en ekstern kunde |
| **Vurdering** | Bokstavkarakter. "Arbeider" (100 %) |

**Organisering:**
- Hver gruppe får **prosjektoppdrag fra en ekstern kunde**
- Obligatorisk oppstart, fellesmøter, forelesninger og **ukentlig veiledning**
- Periodiske møter med gruppeledere
- Hele livssyklusen: forundersøkelse, krav, design, implementasjon, evaluering
- Sluttpresentasjon og demonstrasjon til både kunde og sensor
- Gruppen er ansvarlig for implementasjon inkludert **AI-verktøy**

**Forkunnskaper:** Oppflyttet til 4. årskurs Datateknologi. TDT4140 eller tilsvarende.

**Emneansvarlig:** Maria Letizia Jaccheri, IDI

**Kilde:** [ntnu.no/studier/emner/TDT4290](https://www.ntnu.no/studier/emner/TDT4290)

## IPR / Åndsverksavtaler — NTNUs rammeverk

**✅ Bekreftet:**

| Tema | Status | Regel |
|:-----|:-------|:------|
| **Hovedregel: Studenten eier sin oppgave** | ✅ Bekreftet | Studenten har opphavsrett til egen oppgave. Veiledning gir **ikke** medforfatterskap |
| **NTNUs bruk av oppgaven** | ✅ Bekreftet | Kan bruke til undervisning/forskning, **ikke kommersielt**. Publisering krever samtykke |
| **Utsatt offentliggjøring (båndlegging)** | ✅ Bekreftet | Kan avtales ved samarbeid med bedrift, før innlevering |
| **Standard samarbeidsavtale** | ✅ Bekreftet | Eget standardavtale-dokument for samarbeid mellom student og bedrift |
| **Taushetsplikt (konfidensialitet)** | ✅ Bekreftet | Egen konfidensialitetsavtale tilgjengelig |
| **IP-rettigheter til NTNU** | ✅ Bekreftet | "Erklæring om overføring av IP-rettigheter" — brukes ved EU-/Forskningsrådsprosjekter |

**TDT4290** eksplisitt refererer til NTNUs IPR-rammeverk på emnesiden.

**IT2901** nevner ikke IPR eksplisitt på emnesiden, men som NTNU-emne med eksterne oppdragsgivere vil standardavtalen trolig gjelde (antagelse).

## Hva Syntax & Flow kan lære fra NTNU

| Lærdom | Detaljer |
|:-------|:---------|
| **1. Standardiserte avtaler** | NTNU har ferdige maler for samarbeidsavtale, konfidensialitet og IPR-overføring — å ta utgangspunkt i disse kan spare mye juridisk arbeid |
| **2. Rettigheter: studenten eier som standard** | Utgangspunktet er at studenten/utvikleren eier resultatet. Dette bør speiles i S&Fs avtaler med kunder |
| **3. Båndlegging/utsatt publisering** | Brukes for å gi bedrifter trygghet uten at studenter gir fra seg rettigheter |
| **4. Delt vurdering på gruppenivå** | Begge emnene gir normalt samme karakter til alle i gruppen, med mulighet for individuell justering |
| **5. Ukentlig veiledning og statusrapporter** | Tydelig struktur med faste leveranser og møter gir forutsigbarhet for både studenter og kunder |
| **6. 15 SP per emne** | NTNU bruker 15 SP per prosjektemne — S&F bør vurdere samme størrelse for fleksibilitet |
| **7. AI-verktøy inkluderes** | TDT4290 eksplisitt nevner at studentene står fritt til å velge verktøy, inkludert AI |
