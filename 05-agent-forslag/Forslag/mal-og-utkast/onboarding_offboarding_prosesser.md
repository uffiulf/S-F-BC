# Onboarding- og offboarding-prosesser — Syntax & Flow

---

## A. Onboarding av nye studenter

### Uke 1: Før oppstart
- [ ] Signer studentavtale (emnetilknytning)
- [ ] Få tilgang til Slack, GitHub, Figma, portalen
- [ ] Les Syntax & Flow-operasjonsmanual
- [ ] Sett opp timeføringsverktøy
- [ ] Avklar individuelle læringsmål med mentor

### Uke 2: Første uke i team
- [ ] Bli introdusert for teamet
- [ ] Delta i kundemøte som observatør
- [ ] Gjennomgå prosjektets status og backlog
- [ ] Få tildelt første oppgave

### Uke 3–4: I gang
- [ ] Gjennomfør første selvstendige oppgave
- [ ] Delta i refleksjonsseminar
- [ ] Skriv første refleksjonsnotat

---

## B. Onboarding av nye kunder

### Før oppstart
- [ ] Send og få signert samarbeidsavtale
- [ ] Gjennomfør forventningsavklaringsmøte (30 min)
- [ ] Avklar ukentlig møtetidspunkt
- [ ] Gi kunden tilgang til statusportal (hvis aktuelt)
- [ ] Send velkomst-e-post med kontaktpersoner og møteplan

### Første uke
- [ ] Gjennomfør oppstartsmøte med hele teamet
- [ ] Bli enige om prosjektets omfang og leveranser
- [ ] Avklar kommunikasjonskanaler (e-post, Slack, møter)
- [ ] Avklar taushetsplikt og konfidensialitet
- [ ] Innhent samtykke for porteføljepublisering

### Løpende
- [ ] Ukentlig statusmøte (15–30 min)
- [ ] Månedlig fremdriftsrapport

---

## C. Offboarding av studenter (ved slutten av semesteret)

### Siste måned
- [ ] Gjennomfør sluttvurdering med mentor
- [ ] Skriv prosjektrapport og refleksjonsnotat
- [ ] Overfør kode/dokumentasjon til felles repository
- [ ] Dokumenter passord, tilganger og nøkkelinformasjon
- [ ] Identifiser arvtaker i teamet

### Siste uke
- [ ] Hold overleveringsmøte med arvtaker
- [ ] Gå gjennom «hva du bør vite»-liste
- [ ] Slett personlige tilganger (GitHub, Slack, portalen)
- [ ] Lever inn ev. utstyr
- [ ] Samtykke for porteføljepublisering bekreftes/fornyes

### Etter avslutning
- [ ] Send evaluering til studenten (KPI-måling)
- [ ] Oppdater alumni-database
- [ ] Inviter til alumni-nettverk

---

## D. Offboarding av kunder (ved prosjektslutt)

### Siste uke
- [ ] Gjennomfør overleveringsmøte
- [ ] Lever sluttprodukt (kode, design, rapport) i endelig versjon
- [ ] Send kundeevalueringsskjema
- [ ] Innhent samtykke for porteføljepublisering (hvis ikke allerede gjort)
- [ ] Avtal eventuell videre oppfølging (Betinget SLA) eller avslutning

### Etter prosjekt
- [ ] Arkiver avtaler, rapporter og samtykker
- [ ] Publiser case study på portalen (etter samtykke)
- [ ] Legg til kunde i referansenettverk (hvis aktuelt)
