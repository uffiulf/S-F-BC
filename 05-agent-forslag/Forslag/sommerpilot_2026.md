# Sommerpilot 2026 — Syntax & Flow
> 3-ukers intern pilot. Frivillig — ingen lønn, ingen studiepoeng, ingen eksterne kunder.
> Målet er å teste om konseptet fungerer og bygge teamet før høsten.

---

## Hva studenten får ut av dette

- **Reell prosjekterfaring** — jobbe i team, bruke verktøy, møte deadlines
- **Porteføljeinnhold** — nettside, design, analyse som kan vises frem
- **Tverrfaglig samarbeid** — jobbe på tvers av IT, design og økonomi
- **God referanse** — fra prosjekteier etter fullført pilot
- **Først tilgang** til Syntax & Flow når det formelle kommer på plass til høsten

## Praktisk

| Hva | Detalj |
|-----|--------|
| **Tidsrom** | 3 uker — [dato] til [dato] |
| **Innsats** | 5–10 timer/uke per person (fleksibelt) |
| **Møter** | Fast dag/klokkeslett hver uke fysisk på HiØ (f.eks. mandag 10–12) |
| **Arbeid** | Gjøres på egen hånd mellom møtene |
| **Verktøy** | Discord for daglig kommunikasjon, Trello/GitHub Projects for oppgaver |

---

## Møtestruktur og oppfølging

### Før oppstart (1 samtale per deltaker)
Prosjekteier ringer/snakker med hver deltaker (15 min):
- Hvorfor vil du være med?
- Hva vil du lære?
- Hvor mange timer kan du legge inn i uken?
- Er det greit at det er frivillig uten lønn/studiepoeng?

### Ukentlige møter (fast dag, 2 timer)
Hvert møte følger samme agenda:

| Tid | Aktivitet |
|-----|-----------|
| 10:00–10:15 | Sjekk-in: hva har du gjort siden sist? (alle sier 1–2 setninger) |
| 10:15–10:45 | Status per arbeidspakke — hva er gjort, hva gjenstår, hva er blokkert? |
| 10:45–11:15 | Prosjekteier gir tilbakemelding, justerer prioriteringer, hjelper med blokkeringer |
| 11:15–11:30 | Plan for kommende uke — hvem gjør hva, når er det klart? |
| 11:30–12:00 | Evt. felles arbeidsøkt (parprogrammering, design-review, felles problemløsning) |

### Mellom møtene
- Discord for raske spørsmål og avklaringer
- Prosjekteier sender en kort oppsummering dagen før hvert møte

## Hva studenten må levere

**Én ting per uke.** Ikke mer. Det skal være overkommelig:

| Uke | Leveranse | Hva det inneholder | Godkjent når |
|-----|-----------|-------------------|--------------|
| **1** | Et utkast / en plan | Skisser, research, struktur | ✅ Vises på møte |
| **2** | En prototype / første versjon | Noe som kan vises og testes | ✅ Vises på møte |
| **3** | Ferdig leveranse | Klart til å vises frem | ✅ Presenteres siste møte |

Hver deltaker velger én arbeidspakke ved oppstart og holder seg til den gjennom hele piloten.

## Uke-for-uke-plan (prosjekteier)

### Før uke 1 (forberedelser)

| Dag | Hva |
|-----|-----|
| Mandag | Ring hver deltaker (15 min) — få ja/nei, avklar forventninger |
| Tirsdag | Opprett Discord-server, Trello-board, GitHub-repo |
| Onsdag | Book grupperom på HiØ (samme tid alle 3 ukene) |
| Torsdag | Send velkomst-e-post med praktisk info, møtetid, link til Discord |
| Fredag | **Kickoff-møte (2 timer)** — bli kjent, visjon, velg arbeidspakker, lag team-charter |

### Uke 1

| Dag | Hva |
|-----|-----|
| Mandag | **Møte (2 timer)** — sjekk-in, status, tilbakemelding, plan |
| Tirsdag–torsdag | Svar på spørsmål på Discord. Send påminnelse om møte dagen før |
| Fredag | Sjekk inn: "Hvordan går det? Noe du trenger hjelp med?" |

### Uke 2

| Dag | Hva |
|-----|-----|
| Mandag | **Møte (2 timer)** — sjekk-in, gjennomgang av prototyper, justering |
| Tirsdag–torsdag | Oppfølging på Discord |
| Fredag | Kort sjekk-inn |

### Uke 3

| Dag | Hva |
|-----|-----|
| Mandag | **Møte (2 timer)** — siste gjennomgang før ferdigstillelse |
| Tirsdag–onsdag | Ferdigstillelse og polering |
| Torsdag | Forbered presentasjon |
| Fredag | **Sluttmøte (2 timer)** — presentasjon av leveranser + retrospektiv (hva lærte vi?) |

## Hva kan gå galt — og hva gjør vi da?

| Problem | Løsning |
|---------|---------|
| En deltaker slutter | Fordel oppgavene på resten av teamet. Ingen enkeltperson er kritisk |
| Noen leverer ikke | Snakk med vedkommende på møtet. Juster forventninger eller bytt oppgave |
| Vi blir ikke ferdige | Det er greit — målet er læring, ikke perfeksjon. Dokumenter status |
| Uenighet i teamet | Prosjekteier tar en prat med de det gjelder. Finn en løsning alle kan leve med |
| Noen vil ha lønn/poeng | Minn på at dette er frivillig egenutvikling. Hvis de ikke vil være med, er det greit |
